# Multi-environment Ansible Configuration

This directory contains Ansible configurations for deploying the application across different environments.

## Directory Structure

```
ansible/
├── ansible.cfg                    # Ansible configuration
├── playbook.yml                   # Main deployment playbook
├── verify.yml                     # Verification playbook
├── group_vars/
│   └── all.yml                    # Global variables (auto-generated)
├── inventory/
│   ├── dev/
│   │   ├── hosts                  # Dev inventory (auto-generated by Terraform)
│   │   └── group_vars/
│   │       └── all.yml            # Dev-specific variables
│   ├── staging/
│   │   ├── hosts
│   │   └── group_vars/
│   │       └── all.yml
│   └── production/
│       ├── hosts
│       └── group_vars/
│           └── all.yml
└── templates/
    ├── app.service.j2             # Systemd service template
    └── nginx.conf.j2              # Nginx configuration template
```

## Usage

### Deploy to Development
```bash
ansible-playbook -i inventory/dev/hosts playbook.yml
```

### Deploy to Staging
```bash
ansible-playbook -i inventory/staging/hosts playbook.yml
```

### Deploy to Production
```bash
ansible-playbook -i inventory/production/hosts playbook.yml
```

### Verify Deployment
```bash
ansible-playbook -i inventory/<env>/hosts verify.yml
```

## Environment-Specific Configuration

Each environment has its own configuration in `inventory/<env>/group_vars/all.yml`.

These files are automatically generated by Terraform with environment-specific values.
